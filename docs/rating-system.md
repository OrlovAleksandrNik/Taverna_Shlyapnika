# Система рейтинга

Рейтинг игроков состоит из публичной страницы `rating.html`, API `GET /api/rating`, таблиц `RatingPlayer`, `RatingPlayedGame`, `RatingEvent` и административного меню Telegram-бота.

## Публичные поля

Сайт показывает:

- место игрока;
- публичное имя;
- никнейм, если указан;
- количество сыгранных игр;
- общее количество очков;
- вдохновение;
- среднее количество очков за игру;
- дату последнего обновления статистики.

Приватные Telegram ID администраторов и служебные поля не отдаются в публичном API.

## Сортировка

Порядок рейтинга:

1. `totalPoints` по убыванию;
2. средние очки за игру по убыванию;
3. количество игр по убыванию;
4. имя игрока по алфавиту;
5. `id` как стабильный технический tie-breaker.

## Событийная модель

Очки и вдохновение записываются как события `RatingEvent`. После события агрегаты игрока пересчитываются из журнала, чтобы итог можно было восстановить.

Типы событий:

- `player_created`;
- `player_updated`;
- `player_hidden`;
- `player_shown`;
- `game_result`;
- `points_adjustment`;
- `inspiration_adjustment`;
- `correction`.

## Защита от дублей

`RatingEvent.idempotencyKey` уникален. Если обработчик передаёт один и тот же ключ повторно, база не создаст второе событие.

## Скрытие игроков

Игрок может быть скрыт из публичного рейтинга без удаления истории. Это меняет `isVisible` и создаёт событие в журнале.

## Управление

Изменения выполняются через Telegram-бота администратором: команда `/rating` или кнопка «Рейтинг» в главном меню.
