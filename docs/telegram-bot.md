# Telegram-бот

Бот находится в `src/bot/index.ts` и использует grammY.

## Настройка

1. Создайте бота через BotFather.
2. Укажите токен в `.env`:

```env
TELEGRAM_BOT_TOKEN=
BOT_MODE=polling
ADMIN_TELEGRAM_IDS=
```

Не храните реальный токен в Git.

## Запуск

```bash
pnpm dev:bot
```

Или вместе с API и сайтом:

```bash
pnpm dev
```

## Режимы

- `polling` — локальная разработка. Перед стартом бот удаляет старый webhook.
- `webhook` — production при наличии публичного HTTPS URL и `WEBHOOK_URL`.

## Команды

- `/start` — регистрация или вход мастера.
- `/rating` — меню рейтинга для администратора.
- `/admin` — краткая административная статистика.

## Сценарии мастера

- регистрация мастера;
- создание игры для афиши;
- загрузка изображения игры;
- просмотр своих игр;
- отмена игры;
- изменение базовых полей игры.

## Сценарии администратора рейтинга

- добавить игрока;
- записать сыгранную игру;
- скорректировать очки;
- добавить или снять вдохновение;
- скрыть или вернуть игрока в публичный рейтинг;
- посмотреть историю.

## Доступ

Административные действия доступны мастерам с ролью `admin` или Telegram ID из `ADMIN_TELEGRAM_IDS`.

## Хранение состояния

Состояние диалогов хранится в таблице `BotSession`, поэтому черновики не завязаны только на память процесса.

## Логи

Бот логирует запуск, режим работы, полученные update и ошибки Telegram API. Токен редактируется из сообщений ошибок перед записью в лог.
