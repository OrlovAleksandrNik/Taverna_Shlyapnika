# Telegram-бот

Основной бот мастеров находится в `apps/telegram-bot-java` и работает на Spring Boot через Telegram long polling.

## Настройка

1. Создайте бота через BotFather.
2. Укажите токен в `.env`:

```env
TELEGRAM_BOT_TOKEN=
BOT_MODE=polling
ADMIN_TELEGRAM_IDS=
ADMIN_TELEGRAM_USERNAMES=MisterHatter
BOT_DISPLAY_NAME=Писарь Таверны
BOT_CACHE_CHAT_ID=
BOT_CLEANUP_DELAY_SECONDS=86400
```

Не храните реальный токен в Git.

## Запуск

```bash
pnpm dev:bot:java
```

Или вместе с API и сайтом:

```bash
pnpm local:java:bot
```

## Режимы

- `polling` — локальная разработка. Перед стартом бот удаляет старый webhook.
- `webhook` — production при наличии публичного HTTPS URL и `WEBHOOK_URL`.

## Команды

- `/start` — регистрация или вход мастера.
- `/rating` — меню рейтинга для администратора.
- Нижнее меню закрепляется в чате после первого `/start` и дублирует основные действия: создание игры, список игр, галерея и рейтинг. До первого сообщения Telegram показывает стандартную кнопку Start и меню команд, которое бот настраивает при запуске.

## Сценарии мастера

- регистрация мастера;
- создание игры для афиши;
- просмотр своих игр;
- отмена игры;
- изменение базовых полей игры.

## Сценарии администратора рейтинга

- добавить игрока;
- записать сыгранную игру;
- скорректировать очки;
- добавить или снять вдохновение;
- скрыть или вернуть игрока в публичный рейтинг;
- посмотреть историю.

## Доступ

Административные действия доступны мастерам с ролью `admin`.

Если указан `ADMIN_TELEGRAM_IDS` или `ADMIN_TELEGRAM_USERNAMES`, подходящий мастер автоматически получает роль `admin` при регистрации или следующем обращении к backend. По умолчанию в админах указан `MisterHatter`. Если оба списка пустые, все активные мастера внутреннего бота получают доступ к рейтингу.

## Чистота чата

`BOT_CLEANUP_DELAY_SECONDS` задаёт, через сколько секунд бот попробует удалить свои служебные сообщения из личного чата. Значение `86400` означает примерно сутки. Если нужно отключить удаление, поставьте `0`.

`BOT_CACHE_CHAT_ID` включает кэш-группу: бот будет копировать туда свои служебные ответы перед удалением из личного чата. Чтобы включить этот режим, создайте отдельную группу, добавьте туда бота и укажите ID группы в переменной.

Отображаемое имя бота задаётся через `BOT_DISPLAY_NAME`; по умолчанию используется `Писарь Таверны`. Username бота меняется только через BotFather.

## Хранение состояния

Состояние диалогов хранится в таблице `BotSession`, поэтому черновики не завязаны только на память процесса.

## Логи

Бот логирует запуск, режим работы, полученные update и ошибки Telegram API. Токен редактируется из сообщений ошибок перед записью в лог.
