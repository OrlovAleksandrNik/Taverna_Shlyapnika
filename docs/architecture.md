# Архитектура проекта

«Таверна Шляпника» сейчас устроена как компактный Node/TypeScript-проект с публичным статическим сайтом в корне и backend-слоем в `src`.

## Основной поток данных

```text
Telegram-бот мастера
  -> Prisma Client
  -> PostgreSQL
  -> Express API
  -> script.js
  -> карточки афиши на сайте
```

Бот не редактирует HTML и JavaScript вручную. Он создаёт записи в базе, а сайт получает их через API.

## Слои

- `*.html`, `styles.css`, `script.js` — публичный frontend без сборщика.
- `src/api` — Express-приложение, HTTP-маршруты, CORS, health-check и выдача статических файлов.
- `src/bot` — Telegram-бот на grammY, команды, callback-обработчики и состояние диалогов.
- `src/services` — бизнес-логика игр, заявок, согласий, уведомлений, рейтинга и privacy-операций.
- `src/jobs` — фоновые задачи, сейчас архивация прошедших игр.
- `src/utils` — общая работа со временем и валидацией.
- `prisma` — схема базы и миграции.

## Почему нет физического monorepo `apps/*`

Сайт уже работает как набор статических файлов из корня, а backend отдаёт их через `express.static(root)`. Перенос в `apps/website` потребует отдельной миграции всех относительных ссылок, путей к изображениям, страниц мастеров и Dockerfile. Для текущей публикации структура сохранена, но документация разделяет модули логически.

## Runtime

Один production-процесс `dist/index.js` запускает:

- Express API;
- статическую раздачу сайта;
- Telegram-бота, если задан `TELEGRAM_BOT_TOKEN`;
- периодическую архивацию прошедших игр.

Для разработки доступны отдельные команды `pnpm dev:backend` и `pnpm dev:bot`.
