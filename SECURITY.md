# Security Policy

## Сообщение об уязвимостях

Не публикуйте уязвимости, токены, персональные данные и детали эксплуатации в открытых issue.

Контакт для приватного сообщения владельцу проекта пока не заполнен:

`[КОНТАКТ ДЛЯ ВОПРОСОВ БЕЗОПАСНОСТИ]`

## Секреты

Нельзя добавлять в Git:

- `.env`;
- Telegram Bot Token;
- GitHub tokens;
- строки подключения к production-базе;
- `INTERNAL_API_TOKEN`;
- дампы базы;
- реальные заявки пользователей;
- приватные Telegram ID, если владелец не разрешил публикацию.

Если Telegram-токен попал в публичный канал, его нужно немедленно отозвать через BotFather и выпустить новый.

## Поддерживаемые версии

Пока проект находится до первого публичного релиза. Поддерживаемой считается текущая ветка `main`.

## Шрифты и ассеты

Перед публикацией публичного репозитория владелец должен проверить лицензии локальных шрифтов и исходников дизайна. Если распространение запрещено, такие файлы нужно исключить из Git и описать ручную установку.
